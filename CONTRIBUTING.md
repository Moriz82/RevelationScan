# Contributing

Contributions welcome. Bug reports, feature ideas, and PRs all appreciated.

## Reporting bugs

Open an issue with:
- What you ran (command + version / commit SHA).
- What you expected.
- What happened instead (stack trace, logs, screenshot).

## Submitting changes

1. Fork and branch off `main`.
2. Keep changes focused — one logical change per PR.
3. Match the existing code style. If you add a new dependency, say why in the PR description.
4. Update the README or docs if behavior changes.
5. Open the PR against `main`. Describe the "why", not just the "what".

## Scope

This project is for research / authorized engagement use only. Don't open PRs or issues describing how to use it against systems you don't own or have permission to test.
